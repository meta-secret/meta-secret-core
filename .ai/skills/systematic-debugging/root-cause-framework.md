# Root Cause Framework

Always follow this order:

1. Symptoms
   What exactly is failing?

2. Observations
   What hard facts do we have from logs, build output, runtime output, or behavior?

3. Hypotheses
   What are 2-4 plausible root causes?

4. Evidence
   What supports or contradicts each hypothesis?

5. Root Cause Candidate
   Which hypothesis is most likely and why?

6. Minimal Intervention
   What is the smallest meaningful fix or experiment?

7. Verification
   How do we prove the root cause is actually fixed?
